# 画面仕様書

- 版: 第1版（要件定義書 第2版の 7.1 画面一覧 S1〜S10 に対応）
- 位置づけ: 8月MVP実装とデモの基準。デモのフィードバックで更新し、要件第3版（90%）に反映する
- 共通制約: CLAUDE.md 4章「UI設計制約」を正とする。本書では画面ごとの追加事項のみ記す

## 共通仕様（全画面）

- UIテキストはすべて日本語。専門用語には振り仮名または平易な言い換えを添える
- 生徒向け画面: 本文16px以上・ダークテーマ標準・1画面1タスク・ホバー依存禁止・縦スクロール最小化
- 講師向け画面（NearHub）: タップターゲット44px以上・右クリック依存禁止
- エラー表示: 画面上部に赤帯で「何が起きたか＋次にすること」を平易な日本語で表示（例:「送信できませんでした。もう一度『提出する』を押してください」）。エラーコードのみの表示は禁止
- 読込中: 1秒以上かかる処理はスピナー＋文言（「読み込み中…」）を表示
- 完了条件: 新規・変更画面は GOOVIS / Quest 3 / NearHub / モバイルモニターの4条件で表示確認

## S1: 受講生ホーム（今日やること）

- 利用者: 受講生（対面・eL） / 主デバイス: GOOVIS・ブラウザ・iPad
- 目的: ログイン直後に「今日やること」へ迷わず到達させる（KPI#3の起点）

| 表示項目 | 内容 |
|---|---|
| 今日の授業 | 次の授業コマ（日時・コース名）。当日はコマ開始で「授業に入る」ボタンが有効化 |
| やることリスト | 未提出課題を期限昇順で最大5件。各行 = 課題名＋期限＋状態バッジ |
| 続きから | 前回「取組中」だった課題への1タップ導線 |
| 自分の到達度へ | S5への導線（数値のプレビューは出さない。1画面1タスク維持） |

| 操作 | 結果 |
|---|---|
| 課題行をタップ | S2（プロンプト演習）またはS4（Jupyter）へ遷移 |
| 「先生を呼ぶ」（対面のみ表示） | S6の該当タイルが点滅。ボタンは押下後「呼びました」表示に変化（連打防止・30秒で復帰） |

- 例外: 未提出課題0件のとき「ぜんぶ終わっています。おつかれさま！」と教材一覧への導線を表示

## S2: プロンプト演習（出題閲覧・入力・提出）

- 利用者: 受講生 / 主デバイス: 4デバイス＋eL
- 目的: 課題文を見ながらプロンプトを書き、提出する（F3）

| 表示項目 | 内容 |
|---|---|
| 課題文 | 折りたたみ可能なパネル。入力中も先頭行（タイトル）は常時見える |
| プロンプト入力欄 | 残り文字数を入力中常時表示（上限は課題設定値）。超過時は赤字＋提出ボタン無効化 |
| AI実行結果貼り付け欄・ふりかえりメモ欄 | 任意入力。ラベルに「なくても提出できます」を併記 |
| 状態バッジ | 未着手/取組中/提出済/AI採点済/完了/差戻し（差戻し時は講師コメントを本文上部に表示） |

| 操作 | 結果 |
|---|---|
| 「提出する」 | 確認ダイアログ（「提出します。いいですか？」）→ 提出済へ。5分以内かつAI採点開始前なら「取り消して修正」ボタン表示 |
| 「下書き保存」 | 30秒ごとの自動保存＋手動保存。保存時刻を表示 |

- 例外: 期限超過後は入力欄上部に「期限を過ぎています。提出はできますが『遅延』になります」を常時表示
- 境界値: 文字数上限ちょうどは提出可、+1文字で提出不可（E2E境界値パス対象）

## S3: AI講師チャット

- 利用者: 受講生 / 主デバイス: GOOVIS・Quest 3・ブラウザ
- 目的: 質問→AI応答（F2）。フィルタ経由・注記常時表示

| 表示項目 | 内容 |
|---|---|
| 定型注記 | 「AIによる回答です。わからないときは先生に聞いてね」を画面下部に常時固定 |
| 会話履歴 | 当該コース内の自分の履歴のみ。他人の履歴は参照不可 |
| 質問カテゴリ | 課題について/操作方法/その他（既定: その他） |

| 操作 | 結果 |
|---|---|
| 質問送信 | 5秒以内に応答開始。超過で「考え中」＋定型ヒント、10秒でタイムアウト→「もう一度きく」ボタン |
| （フィルタブロック時） | 「この質問にはお答えできません。先生に聞いてください」を表示、講師へ通知 |

- Quest 3表示: 文字サイズをGOOVIS比1.25倍、行間広め。入力は定型ボタン（「ヒントがほしい」「もっとくわしく」等）を優先し、自由入力は補助
- 個人情報検出時: 送信前に「個人情報（名前・電話番号など）は入力しないでね」を表示し、該当部分をマスキングした形で送信

## S4: Jupyter演習

- 利用者: 受講生 / 主デバイス: GOOVIS・ブラウザ
- 目的: CanvasからワンタップでJupyterHubの自分の環境に入る（F2）

| 操作 | 結果 |
|---|---|
| 「演習をはじめる」 | LTI連携でJupyterHubへ（再ログイン不要）。起動待ちは進捗表示（「じゅんび中… あと少し」） |
| 「再起動」 | 自分のカーネルのみ再起動。確認ダイアログ（「書いたコードは残ります」）付き |

- 例外: 校内GPUサーバー（RTX 6000 Ada）停止時は「いま演習マシンが止まっています」＋静的教材モードへの導線を表示（F2例外3）

## S5: 自分の到達度

- 利用者: 受講生 / 主デバイス: 全デバイス
- 目的: 自分の伸びを見て前向きになれる画面（他人との比較は表示しない）

| 表示項目 | 内容 |
|---|---|
| 到達度スコア | 今週の値＋先週からの変化（「先週より +3」）。下降時も否定的表現は使わない（「今週はゆっくりペース」） |
| 週次推移グラフ | 折れ線・大きめ・高コントラスト（Quest 3判読可能な線幅） |
| 出席・提出状況 | 出席率/提出率。「計測不能」週はグレー表示＋「この週は記録がありません」 |

## S6: 授業中モニタリング（16タイル）

- 利用者: 講師 / 主デバイス: NearHub S65（タッチ）
- 目的: 15名の状態を一目で把握し、介入先を決める（F4）

- レイアウト: 4×4格子（教室の座席配置と同じ並び。デバイス割当の座席番号に連動）
- タイル内容: 表示名・座席番号 / 状態色（未着手=灰・取組中=青・提出済=緑・差戻し=橙）/ 停滞バッジ（5分無操作）/ AI質問回数 / ヘルプ点滅
- 教室投影の注意: 本画面はNearHubに表示されるため、**到達度スコアなど個人の成績値はタイルに表示しない**（状態色と停滞のみ）。成績詳細はタイルタップ→講師のみ確認できる詳細パネルで表示

| 操作 | 結果 |
|---|---|
| ヘルプ点滅タイルをタップ | 点滅消込＋対応済みマーク |
| タイル長押し（1秒） | 詳細パネル（現在の入力状況・直近のAI質問・提出履歴）。パネルは「閉じる」ボタンで明示的に閉じる |

- 例外: データ更新が遅延した場合「データ更新遅延中（最終更新 hh:mm）」バナー（F4例外5）

## S7: 採点・差戻し

- 利用者: 講師・管理者 / 主デバイス: 講師機・NearHub
- 目的: AI採点済の提出を確認し、完了 or 差戻しにする（F3）

| 表示項目 | 内容 |
|---|---|
| 採点待ち一覧 | AI採点済の提出を古い順。乖離フラグ（AI/講師差20点以上）は赤マーク |
| 提出詳細 | プロンプト本文・実行結果・ふりかえり・版履歴・AI採点（総合/観点別/講評/採点根拠） |
| 講師スコア入力 | 0-100。既定値=AIスコア（そのまま確定可） |

| 操作 | 結果 |
|---|---|
| 「完了にする」 | 講師確定スコアがCanvas成績表へ反映 |
| 「差戻す」 | コメント必須（最大1,000文字）。未入力時はボタン無効＋理由表示 |
| 遅延提出の「承認して反映」 | 遅延フラグ付きのまま成績反映 |

## S8: 週次到達度レポート

- 利用者: 講師・管理者 / 主デバイス: 講師機・管理者PC
- 内容: クラス別サマリ→受講生別詳細のドリルダウン。停滞アラート（2週連続下降）を最上部に表示
- 印刷/PDF出力可（保護者面談用。氏名は表示名のみ）

## S9: ユーザー管理・デバイス割当

- 利用者: 管理者（デバイス割当は講師も可） / 主デバイス: 管理者PC
- ユーザー管理: CSV取込（要件定義書F1の列定義）＋個別追加。氏名フルネームを表示する唯一の画面（5.3）
- デバイス割当: 座席番号×受講生の割当表。授業コマ単位で付け替え可。割当変更は監査ログ記録

## S10: 監査ログ閲覧

- 利用者: 管理者のみ / 主デバイス: 管理者PC
- 検索条件: 期間・操作者・操作種別・対象エンティティ
- 表示: 変更前後の差分をハイライト。エクスポートはCSV（出力操作自体も監査ログに記録）
- 権限外アクセスは403（講師・受講生・ゲスト。E2E必須）
