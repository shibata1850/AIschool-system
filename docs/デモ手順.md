# デモ手順・使い方ガイド

このシステムが「実際にどう使われるか」を、デモしながら理解するための資料。

## 0. 全体像: 2つのシステムの役割

```
Canvas LMS（土台）                    カスタム層（AIの活動場）
─────────────────                    ──────────────────
・コース／受講生名簿                    ・プロンプト演習（AIに書かせる練習）
・課題（アサインメント）                ・AI講師チャット
・成績表（評定）                       ・到達度ダッシュボード
・出欠（別ツール）                      ・授業中モニタリング（16席）
・ログイン＝本人確認の源泉              ・採点補助・週次レポート
        │                                    ▲
        └────── LTI 1.3で連携（Canvasにログイン→中でカスタム層が開く）──┘
```

**ひとことで**: Canvasが「学校の事務・記録の土台」、カスタム層が「その上に乗るAIの授業活動」。
生徒はCanvasにログインし、コースの中の「AI講師・演習」を開くと、カスタム層の画面が現れる。

## 1. 誰が・いつ・何をするか

### 講師（授業前）
1. **Canvasで**コースを作り、受講生を登録、課題を作る（例: プロンプト演習①）
   → いまはデモデータ投入済み（`プロンプト演習デモ（架空）`＋デモ生徒5名＋課題3件）
2. 教材・スライドを準備

### 生徒（授業中）
1. **Canvasにログイン** → 自分のコースを開く
2. 左メニューの **「AI講師・演習」** をクリック → カスタム層が開く
3. **プロンプト演習**: 課題文を読み、AIへの指示（プロンプト）を書いて「提出する」
4. わからないことは **「AI講師にきく」** で質問（フィルタ経由で安全な回答）
5. **「じぶんの到達度」** で自分の伸びを確認

### 講師（授業中・授業後）
1. **授業中モニタリング**（カスタム層）: 16席の状態を色で把握（未着手/取組中/提出済/差戻し）
2. **採点・差戻し**（カスタム層）: AIの一次採点を確認し、完了 or 差戻し
3. 確定した成績は **Canvasの成績表** に反映される
4. **週次到達度レポート**（カスタム層）: クラス全体の伸び・停滞アラートを確認

## 2. デモの進め方（クリックで体験する）

前提: 実環境に再デプロイ済み（`DEMO_RICH_SEED=1`）＋Canvasにリッチデータ投入済み
（`seed-canvas-rich.sh` 実行済み）。

### A. 生徒の体験を見せる
1. Canvas（`https://canvas.133-125-225-64.sslip.io/`）に管理者でログイン
2. コース『プロンプト演習デモ（架空）』→ 左メニュー **「AI講師・演習」**
3. 開いたカスタム層で:
   - **きょうやること**: 課題が状態バッジ付きで並ぶ（差戻し・取組中）
   - 課題を開く → プロンプト入力・文字数カウント・提出
   - **AI講師にきく**: 質問を入力 → 回答（デモはテスト応答）
   - **じぶんの到達度**: 週次の伸びグラフ

### B. 講師の体験を見せる（同じくカスタム層内）
1. **授業中モニタリング**: 16席が状態色で埋まる（教室の一目把握）
2. **採点・差戻し**: 採点待ちの提出にAI一次採点が付く → 完了/差戻し
3. **週次到達度レポート**: 16人の到達度・停滞アラート
4. **クラス名簿・成績サマリ（Canvas連携）**: Canvasの実データ（デモ生徒5名・成績）

### C. Canvas本体を見せる（土台の確認）
1. Canvasのコース → **課題**: プロンプト演習①②③が並ぶ
2. **評定（成績表）**: 受講生×課題の成績が入っている
3. → 「カスタム層で採点した点数が、この成績表に反映される」関係を説明

## 3. どこがデモ用で、どこが本物か（正直な整理）

| 要素 | 状態 |
|---|---|
| Canvasのコース・名簿・課題・成績 | **実データ**（Canvasに実在。架空の人物） |
| カスタム層の講師画面（名簿・採点入力・成績サマリ） | Canvasの**実データを読む** |
| カスタム層の生徒画面・モニタリング・レポート | **デモデータ**（`DEMO_RICH_SEED`。授業風景の再現） |
| AI講師の回答 | **テスト応答**（実AIは `AI_PROVIDER=claude` 設定時） |

## 4. 本番に向けて残ること（このデモの先）

- 生徒の提出を実際にCanvasの課題へ保存（LTI Advantage / AGS）
- 実受講生の登録・独自ドメイン・Pマーク整合（実データ投入前）
- AI講師を実AI（Claude API）に接続＋未成年フィルタの精緻化
- 機材搬入後（9月）の4デバイス実機確認・16台負荷試験
