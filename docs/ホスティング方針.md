# ホスティング方針（段階移行）

- 決定: 2026-07-04 柴田さま（docs/未決事項.md #2）
- 方針: カスタム層はまず**Vercel**でデモ・レビュー運用し、本格運用の段階で
  **さくらのクラウド**へ移管する。Canvas LMS本体は最初からさくらのクラウドに置く

## 1. 何をどこに置くか

| コンポーネント | デモ期（〜8月デモ） | 本番期（9月〜開校） |
|---|---|---|
| カスタム層（本リポジトリ・Next.js） | Vercel | さくらのクラウド（Docker） |
| Canvas LMS本体（Rails+PostgreSQL+Redis） | さくらのクラウド | さくらのクラウド |
| AI推論・JupyterHub・CloudXR | 校内GPUサーバー（オンプレ） | 校内GPUサーバー（オンプレ） |

**重要: Canvas LMS本体はVercelには載りません。** VercelはNext.js等の
フロントエンド／サーバーレス向けで、Railsの常駐サーバー＋PostgreSQL＋Redisと
いう構成は対象外です。Vercelを使う場合も、Canvasを動かす時点でさくらの
クラウド（または他のサーバー）が別途必要になります。

## 2. Vercel→さくら移管の互換性評価

### 問題にならない点（移管前提で実装済み）

- Vercel固有機能（KV / Blob / Cron / Edge Config等）は**一切使っていない**。
  標準の `next build` / `next start` で動く構成
- 本番ビルド（`next start`）に対してE2E 154件のPASSを継続確認しており、
  これはさくらでの動作形態そのもの
- 権限ガード（proxy.ts）はCookie読み取りのみでEdge互換
- 環境変数は `.env.example` に集約済み。Vercelのダッシュボードでも
  さくらのDocker環境でも同じ変数名で設定するだけ

### 注意が必要な点（Vercel利用時の制約）

1. **インメモリストアはサーバーレスで保持されない（最重要）**
   参照実装はデモデータをメモリに持つ。Vercelはリクエストごとに別インスタンスへ
   割り振られ得るため、「提出→採点→講師確認」の一連の操作デモは
   **データが消える／別インスタンスで見えない**ことがある。
   - Vercelデモの用途は**画面デザイン・文言のレビュー中心**と割り切る
   - 操作フローの通しデモが必要になったら、その時点でさくらに
     Docker 1台を立てるのが早い（それがそのままステージングになる）
2. **バックグラウンドAI採点の打ち切り**
   サーバーレスは応答返却後の処理を保証しない。Vercelで実AI採点まで動かす場合は
   Next.jsの `after()` への載せ替えが必要（さくらの常駐サーバーでは不要）
3. **実行時間の上限**
   Vercelの無料プランは関数の実行時間が短く、実AI応答（数十秒）が
   打ち切られる可能性がある。デモはモックAI（AI_PROVIDER=mock）のままを推奨
4. **個人情報の取り扱い**
   Vercelには実受講生の個人情報を載せない（架空デモデータのみ）。
   国内保管方針・Pマーク整合（要件定義書5.3）は、さくら移行後に満たす

### 「環境変数が必要になったら移行」について

環境変数の設定自体はVercelでも問題なくできる（ダッシュボードで設定）。
実際の移行トリガーは環境変数ではなく、次のいずれかが正確:

- Canvas本体を立てる時（どのみちさくらが必要になる）
- 提出データ等の**永続化**が必要になる時（インメモリ→DB）
- 実受講生データを扱い始める時（国内保管・Pマーク整合）

## 3. 移管手順（さくらのクラウドへ）

1. さくらのクラウドにサーバー確保（Ubuntu + Docker）
2. 本リポジトリの `Dockerfile` でイメージをビルドして起動:
   ```
   docker build -t lms-custom .
   docker run -d -p 3000:3000 --env-file .env lms-custom
   ```
3. Vercelに設定していた環境変数を `.env` へ移す（値はパスワード管理台帳から）
4. リバースプロキシ（nginx等）でTLS終端し、ドメインのDNSを切替
5. E2Eをさくら環境に対して1回実行して受け入れ確認

コードの変更は不要（同じ `next build` 成果物が動く）。
