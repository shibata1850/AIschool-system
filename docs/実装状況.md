## 既知の残課題

1. **二重提出の競合**: 提出APIは非アトミック。状態遷移ガードで実害は限定的だが、
   Canvas接続時に楽観ロック（版数チェック）を入れる
2. **チャットのサーバー側キャンセル**: クライアントは10秒で中断するが、
   実プロバイダの推論は最後まで実行される（ストリーミング化で解消予定）
3. **氏名のマスキング**: パターン検出は連絡先のみ。氏名は辞書/NER等を本番導入時に検討
4. **実機依存の受け入れ項目**: 4デバイス実機表示確認・16台負荷試験・低帯域試験は
   機材搬入後（docs/テスト計画書.md 4〜6章）
